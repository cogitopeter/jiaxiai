📤 两种分发方式
⚙ Claude Code 用户
把 skill名.md 发给对方
对方放入 ~/.claude/commands/
重启 Claude Code
用 /skill名 + 题目图片即可
📦 WorkBuddy 用户
把 workbuddyzip/skill名.zip 发给对方
对方在 WorkBuddy 里安装 zip
用 /skill名 + 题目图片即可
zip 若过期见下方"更新 zip"流程
+ 新增 Skill 完整流程
1
在 ~/Desktop/shuxue/ 新建 新skill名.md
参照现有 md 的格式，必须包含 ## PROBLEM_TEMPLATE 和 ## INDEX_TEMPLATE 两个 section，否则 skill-pack 会跳过这个文件。修改 frontmatter 的 name 和 description，以及工作目录路径（如果换目录）。
2
把 md 文件复制到 ~/.claude/commands/ ← 唯一需要手动做的步骤
这样 Claude Code 才能识别为 slash command。~/Desktop/shuxue/ 里的是"源文件"，~/.claude/commands/ 里的是"生效文件"，两份保持同步。skill-pack 管不到这一步。
3
打包 WorkBuddy zip
在 Claude Code 里确保工作目录是 ~/Desktop/shuxue/，然后执行 /skill-pack。全量处理——不判断新旧，把当前目录所有 .md（除 README.md）全部重打包，同名 zip 直接覆盖，新 skill 的 zip 一并生成。无需手动删旧文件。
4
分发
Claude Code 用户发 .md，WorkBuddy 用户发 workbuddyzip/skill名.zip。
5
（可选）更新 www/ 宣传页
在 www 网站里补充新 skill 的介绍内容。
注意1： /skill-pack 只处理当前目录的 .md 文件，运行时工作目录必须是 ~/Desktop/shuxue/。如果 Claude Code 的 cwd 不对，先 cd ~/Desktop/shuxue 再运行。
注意2： 如果删掉了某个 .md 文件，/skill-pack 不会自动清理对应的旧 zip——它只管"md → zip"，不管反向删除。需要手动去 workbuddyzip/ 删除孤立的旧 zip。
✎ 修改已有 Skill 后的更新步骤
1
修改 ~/Desktop/shuxue/skill名.md
这是源文件，在这里改。
2
同步到 ~/.claude/commands/skill名.md
Claude Code 读的是 commands/ 里的版本，要手动覆盖过去。
3
重跑 /skill-pack 更新 zip
WorkBuddy 用户重新下载新 zip 安装。
